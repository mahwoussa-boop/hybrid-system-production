# الدستور المعماري للنظام الهجين (Hybrid System Architecture Manifesto)

هذا المستند يمثل القواعد الصارمة والقيود المعمارية التي يجب الالتزام بها في جميع عمليات التطوير والصيانة لهذا النظام.

## 1. البنية التحتية المستقلة (Self-Contained Infrastructure)
- **قاعدة البيانات:** يُمنع استخدام خدمات خارجية مثل PostgreSQL أو Redis. يتم الاعتماد حصرياً على **SQLite** المُدارة عبر `utils/db_manager.py`.
- **الطوابير (Queues):** تتم إدارة طوابير المهام داخلياً عبر `asyncio.Queue`. يُحظر إدخال RabbitMQ أو أي وسيط رسائل خارجي.

## 2. حماية الذاكرة (Memory Management)
- يُحظر تخزين البيانات المكشوطة (Scraped Products) في قوائم (Lists) داخل الذاكرة (RAM).
- **القاعدة:** يجب كتابة كل منتج فور استخراجه مباشرة في جدول `raw_scrape_staging` لتجنب فقدان البيانات أو استهلاك الذاكرة.

## 3. حماية المعالج وحلقة الأحداث (CPU & Event Loop Safety)
- يُمنع منعاً باتاً تجميد حلقة الأحداث (Event Loop Blocking).
- أي مكتبة أو عملية تستهلك المعالج بشكل مكثف (مثل معالجة الصور عبر `Pillow`) يجب أن تُنفذ داخل `asyncio.to_thread()`.

## 4. أمان التكامل (Transactional Outbox)
- يُحظر إجراء طلبات HTTP متزامنة أو مباشرة لخدمات خارجية (مثل Make.com) من داخل محركات التسعير.
- **الآلية:** يتم إدراج جميع القرارات والأحداث في جدول `outbox_events` أولاً، وتتولى وحدة `Dispatcher` إرسالها لاحقاً لضمان عدم ضياع البيانات في حال فشل الاتصال.

## 5. خفض التكاليف وحدود الاستخدام (AI Rate Limits)
- جميع استدعاءات نماذج الذكاء الاصطناعي (LLM) يجب أن تمر عبر `match_cache_v2`.
- يجب تنفيذ عمليات المطابقة على دفعات (Batches) مع وجود فاصل زمني `await asyncio.sleep` بين الطلبات لتجنب تجاوز حدود الاستخدام (Rate Limits).
